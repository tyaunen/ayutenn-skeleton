# intro
## ayutennについて
このプロジェクトでは自作のフレームワークayutennを使用しています。
ayutennは、vendor/tyaunen/ayutenn-core に格納されています。

## ディレクトリ構造
* **/app/api** API処理を記述します。
* **/app/controller** コントローラー処理を格納します。
* **/app/database** データベースアクセスを格納します。
* **/app/helper** 繰り返し使用するクラスや関数を格納します。
* **/app/model** コントローラーやAPIがバリデーションを行うための定義ファイル(モデルファイル)を格納します。
* **/app/public** エントリポイントであるindex.phpと、imgやcss、jsといったアセットを格納します。
* **/app/routes** ルーターのためのルート定義を格納します。
* **/app/routes/middleware** ログインチェックなど、ルート定義のためのミドルウェアを格納します。
* **/app/views** HTMLの記述を格納します。
  * **/app/views/compornents** 切り分けて外出しされたHTMLのパーツや、それを出力するクラスを格納します。
  * **/app/views/compornents/flat** Componentのうち、クラスではないファイルを格納します。
* **/app/config** 設定ファイルを格納します。
  * **/app/config/config.json** 環境によって異なる設定を記述します。
  * **/app/config/app.json** このプロジェクト固有の設定を記述します。どの環境でも同じ内容です。


## 処理の流れ
### エントリポイント
ユーザーのリクエストは、まず/app/public/index.phpで受け取ります。
index.phpは定数定義と初期設定、セッションやクッキーの設定を行ったあとで
ayutenn\core\routing\Routerを使用して、URLをもとに適切なファイルへ処理を移譲します。

### Router
Routerはまず、app/routes/*.phpをルート定義ファイルと認識してすべて読み込みます。
それから、リクエスト情報をルート定義と突き合わせて適切な処理を行います。

この処理は4種類に分類できます。
API、コントローラー、ビュー表示、そしてリダイレクトです。

### API
APIは、モデルファイルを使ってリクエストパラメタをバリデートし、何らかの処理を行ったあとでjson形式でレスポンスを返します。

### コントローラー
コントローラーは、モデルファイルを使ってリクエストパラメタをバリデートし、何らかの処理を行ったあとで指定したパスにリダイレクトします。

### ビュー
ビューは、HTMLを表示します。

### リダイレクト
別のパスにリダイレクトします。
つまり、エントリポイントからまた処理を繰り返します。

## 開発の進め方
ファイル個別の書き方は、このファイルと同じディレクトリにある別のドキュメントを参照してください。
